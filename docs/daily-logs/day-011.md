### Step 2: Build native_sim DUT

Problem 1: default `west build -b native_sim` pulls sysbuild → CMake error.
Fix: `--no-sysbuild` (native_sim is single-image).

Problem 2: First try set `CONFIG_BT_USERCHAN=y` directly → Kconfig error
"not directly user-configurable (has no prompt)".
Fix: Delete that line. `native_sim.dts` already has
`chosen { zephyr,bt-hci = &bt_hci_userchan; }` which auto-selects it via
`DT_HAS_ZEPHYR_BT_HCI_USERCHAN_ENABLED`.

Final overlay `scripts/native-userchan/native.conf`:
```
CONFIG_SPI_NRFX=n
CONFIG_USE_SEGGER_RTT=n
CONFIG_LOG_BACKEND_RTT=n
```

Build:
```
west build -b native_sim -d build-native --no-sysbuild \
  -- -DEXTRA_CONF_FILE=scripts/native-userchan/native.conf
```
→ `build-native/zephyr/zephyr.exe` (2.4 MB), clean.

## Trade-offs / Known Limitations

- **Two HCI adapters required**: hci0 owned by BlueZ/Chrome host; hci1
  claimed exclusively by DUT via HCI_CHANNEL_USER.
- **setcap is local, not committed**: `build-native/zephyr/zephyr.exe` is
  gitignored (`build-*/`). On fresh clone the interviewer needs to rebuild
  + re-run setcap. Documented in `scripts/native-userchan/run.sh` header.
- **Device chooser manual pick**: Chromium's native BLE chooser is modal
  and cannot be scripted from the page context (Chrome security). User
  clicks "Connect" then picks `nRF5340_HR` from the chooser — this is the
  skill's documented semi-auto pattern.
- **`BT_USERCHAN` is NOT set via .conf**: must be auto-selected by
  `native_sim` DT chosen. Any attempt to set it directly fails with
  "no prompt" error.
