- `brew install nrfutil` — Homebrew cask，装到 `/opt/homebrew/bin/nrfutil` (v8.1.1)

- `west zephyr-export` 注册 CMake 包

### 任务 3: 修复 — sdk-nrf 路径必须叫 `nrf`
- **坑**: 默认 `repo-path: sdk-nrf` 会把项目 clone 到 `./sdk-nrf/`
- NCS 内部多处硬编码 `${ZEPHYR_BASE}/../nrf/...`（如 `sdk-nrf/subsys/bootloader/Kconfig:45`），导致：
  - `include(ncs_sysbuild_extensions)` 找不到
  - `${ZEPHYR_NRF_MODULE_DIR}` 保持字面未展开
  - `zephyr_modules.txt` 空
- **解法**: `west.yml` 里加 `path: nrf`（显式指定 workspace 内路径）
- `rm -rf sdk-nrf && west update` → NCS 落到 `/Users/rjwang/fun/nrf/` ✅

### 任务 4: 修复 — prj.conf Kconfig 错误
- **坑 1**: `CONFIG_BT_HCI_IPC=y` 不是用户可配置 — nRF5340 board defconfig 通过 `chosen { zephyr,bt-hci }` 自动 select。移除之。
- **坑 2**: `CONFIG_BT_DEVICE_APPEARANCE=833  # Generic Heart Rate Sensor` — int 型符号不能有行尾注释。移除之。

### 任务 5: 修复 — 源码 API 不匹配
- **坑 1**: `BT_LE_ADV_CONN_FAST_1` 是 Zephyr 4.x 的宏；NCS v2.9.0 用 Zephyr v3.7.99-ncs2。改用 `BT_LE_ADV_CONN`。
- **坑 2**: `SPI_DT_SPEC_GET(DT_NODELABEL(spi4), ...)` 需要一个**子节点**（slave device）才能求值（需 `spi-max-frequency`、`duplex`、`frame-format`）。 
  - 对无 CS 的纯 wire-loopback 场景，最干净的方案：
    - `DEVICE_DT_GET(DT_NODELABEL(spi4))` 拿到 bus controller
    - 手动构造 `struct spi_config`（`.cs = {0}`，`.frequency` 从 DT `clock-frequency` 取）
    - 改用 `spi_transceive()` / `device_is_ready()` 的旧 API

### 技术笔记 (坑 + 教训)
- **NCS v2.9.0 强制 workspace 路径**：sdk-nrf 必须叫 `nrf`。`west.yml` 里 `path:` 是必填（不是默认从 `repo-path` 推导）。
- **nrfutil env 脚本化**：优先 `--as-script > file && source file`，避免 zsh quoting 问题。
- **Zephyr 版本差异**：NCS v2.9.0 = Zephyr v3.7.99-ncs2。很多新 BLE adv 宏（`BT_LE_ADV_CONN_FAST_*`）只在 Zephyr 4.x 主线有；稳妥起见用老 API `BT_LE_ADV_CONN` / `BT_LE_ADV_CONN_NAME`。
- **SPI_DT_SPEC vs DEVICE_DT_GET**：无 slave 的 loopback 场景用 `DEVICE_DT_GET + spi_config` 更干净；`SPI_DT_SPEC_GET` 需完整 child node（`compatible`/`reg`/`spi-max-frequency`/`duplex`/`frame-format`）。
- **toolchain 安装路径**：Mac 上 nrfutil 走 Homebrew cask → `/opt/homebrew/bin/nrfutil`；toolchain 本体在 `/opt/nordic/ncs/toolchains/<hash>/`，sdk 在 `<hash>/opt/zephyr-sdk`。
