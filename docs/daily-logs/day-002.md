### Task 4: fix — prj.conf Kconfig errors
- **Gotcha 1**: `CONFIG_BT_HCI_IPC=y` is not user-configurable — on nRF5340 the board defconfig auto-selects it via `chosen { zephyr,bt-hci }`. Remove it.
- **Gotcha 2**: `CONFIG_BT_DEVICE_APPEARANCE=833  # Generic Heart Rate Sensor` — int-typed symbols cannot have a trailing comment. Remove it.

### Task 5: fix — source-code API mismatches
- **Gotcha 1**: `BT_LE_ADV_CONN_FAST_1` is a Zephyr 4.x macro; NCS v2.9.0 uses Zephyr v3.7.99-ncs2. Use `BT_LE_ADV_CONN` instead.
- **Gotcha 2**: `SPI_DT_SPEC_GET(DT_NODELABEL(spi4), ...)` needs a **child node** (slave device) to evaluate (requires `spi-max-frequency`, `duplex`, `frame-format`).
  - For a CS-less pure wire-loopback, the cleanest path:
    - `DEVICE_DT_GET(DT_NODELABEL(spi4))` for the bus controller
    - manually build `struct spi_config` (`.cs = {0}`, `.frequency` from the DT `clock-frequency`)
    - switch to the older `spi_transceive()` / `device_is_ready()` API

### Technical notes (gotchas + lessons)
- **NCS v2.9.0 forces the workspace path**: sdk-nrf must be named `nrf`. `path:` in `west.yml` is mandatory (not derived from `repo-path` by default).
- **Script nrfutil env output**: prefer `--as-script > file && source file` to avoid zsh quoting issues.
- **Zephyr version differences**: NCS v2.9.0 = Zephyr v3.7.99-ncs2. Many new BLE adv macros (`BT_LE_ADV_CONN_FAST_*`) only exist on Zephyr 4.x mainline; play it safe with the older `BT_LE_ADV_CONN` / `BT_LE_ADV_CONN_NAME`.
- **SPI_DT_SPEC vs DEVICE_DT_GET**: without a slave node, the loopback scenario is cleaner via `DEVICE_DT_GET + spi_config`; `SPI_DT_SPEC_GET` needs a complete child node (`compatible`/`reg`/`spi-max-frequency`/`duplex`/`frame-format`).
- **Toolchain install path**: on Mac, nrfutil uses Homebrew cask → `/opt/homebrew/bin/nrfutil`; the toolchain itself is in `/opt/nordic/ncs/toolchains/<hash>/`, and the SDK in `<hash>/opt/zephyr-sdk`.
