**Gotcha #1: bundled git depends on `libunistring.so.2` (Ubuntu 24.04 ships only libunistring.so.5)**
- After `source /tmp/ncs_env.sh`, `LD_LIBRARY_PATH` points at NCS bundled libs, so system git-remote-https is polluted at runtime (NCS's libcurl linked libunistring.so.2)
- Fix: `unset LD_LIBRARY_PATH GIT_EXEC_PATH GIT_TEMPLATE_DIR`, `export PATH=/usr/bin:$PATH` to force system git 2.43 + system libs
- First pass mbedtls download failed (suspected network blip), second west update resumed and succeeded

### 5. west build --sysbuild (first Linux build) ✅
**Gotcha #2: system cmake exits 127 under NCS `LD_LIBRARY_PATH`**
- Tried: unset LD_LIBRARY_PATH + use NCS cmake — works, but requires overriding envs one by one
- **Final approach**: `nrfutil toolchain-manager launch --ncs-version v2.9.0 --shell -- <cmd>` — the officially provided sandbox, auto-isolates env while loading all toolchain deps correctly
- All subsequent build/verify commands go through the `nrfutil ... launch -- <cmd>` template

### Key learnings (write into memory)
1. **nrfutil official URL changed**: the Linux binary moved from `linux-amd64/nrfutil` to `x86_64-unknown-linux-gnu/nrfutil` — migration-handoff §2 needs an update
2. **NCS LD_LIBRARY_PATH pollutes system git/cmake**: Ubuntu 24.04 + NCS toolchain interaction gotcha. **Idiomatic workaround**: wrap all `west` commands in `nrfutil toolchain-manager launch --ncs-version v2.9.0 --shell -- <cmd>` to avoid manual env juggling
3. **Board-specific overlay is a precondition for bsim / multi-board scenarios**: top-level `app.overlay` vs `boards/<board>.overlay`; the former applies to every board — hardware-only nodes (spi/uart/gpio) are safer in the latter
4. Linux cross-platform build has zero feature regression — every prj.conf/DTS/C decision from Day 1–4 is portable
