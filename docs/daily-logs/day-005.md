### 4. west init + update ⚠️→✅
工作区布局: `~/fun/` 作为父目录（与 macOS `~/ncs-ws/` 角色等价），`.west/` 落在 `/home/rjwang/fun/.west/`，`nrf/`/`zephyr/`/`nrfxlib/` 作为 `Nordic_nRF5340_SPI_loopback/` 的 siblings。

**坑 #1: 打包 git 依赖 `libunistring.so.2`（Ubuntu 24.04 只有 libunistring.so.5）**
- `source /tmp/ncs_env.sh` 后 `LD_LIBRARY_PATH` 指向 NCS bundled libs，系统 git-remote-https 运行时也被这些 libs 污染（NCS 的 libcurl 链接了 libunistring.so.2）
- 修复: `unset LD_LIBRARY_PATH GIT_EXEC_PATH GIT_TEMPLATE_DIR`，`export PATH=/usr/bin:$PATH` 强制用系统 git 2.43 + 系统 libs
- 首次跑 mbedtls 下载失败（疑网络瞬断），第二次 west update 续跑成功

### 5. west build --sysbuild (Linux 首次构建) ✅
**坑 #2: 系统 cmake 在 NCS `LD_LIBRARY_PATH` 下异常退出 127**
- 尝试方案: unset LD_LIBRARY_PATH + 使用 NCS cmake — 可行但需逐一覆盖 env
- **最终方案**: `nrfutil toolchain-manager launch --ncs-version v2.9.0 --shell -- <cmd>` — 官方提供的 sandbox，自动隔离 env，同时正确加载 toolchain 所有依赖
