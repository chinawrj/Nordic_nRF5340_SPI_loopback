### 4. Look at GitHub Actions CI ⚠️ → fix ✅
**Two runs both FAILED** (runs `24654121778` + `24655705928`).

Root cause from logs:
```
2026-04-20T08:12:12.354Z /home/runner/.local/bin/nrfutil: line 5: syntax error near unexpected token ']'
2026-04-20T08:12:12.354Z /home/runner/.local/bin/nrfutil: line 5: `  } ]'
```

**Exact same gotcha I hit locally on Day 5**: the workflow used the old URL `linux-amd64/nrfutil` → artifactory returns a 404 JSON body, and `curl -sL` does not detect HTTP errors, so it downloads the JSON as "nrfutil", then bash tries to execute it and crashes.

**Fix `.github/workflows/build.yml` in four places**:
1. **Install step uses the new URL** — `x86_64-unknown-linux-gnu/nrfutil`; add `-f` so HTTP errors exit non-zero
2. **Cache path fixed** — changed from `/opt/nordic/ncs/toolchains` (macOS habit) to `~/ncs/toolchains` + `~/.nrfutil` (real Linux path, Day 5 verified); bumped cache key suffix to `-v2` so old cache is invalidated
3. **Env propagation completed** — added `LD_LIBRARY_PATH` / `PYTHONPATH` / `PYTHONHOME` to the `GITHUB_ENV` exports (required at runtime by NCS python 3.12 + bundled libcurl)
4. **apt prereqs trimmed** — removed `gcc-multilib/g++-multilib/libmagic1` (Day 5 confirmed 404 on noble security mirror; ARM target does not need multilib; libmagic1 renamed libmagic1t64 and comes in transitively)

### CI fix iterations (Day 6's hardest fight)

From Day 5 commit push to CI green: **3 workflow fixes + 4 CI runs**:

| Run | ID | Result | Cause | Fix commit |
|-----|----|--------|-------|-----------|
| #1 | 24654121778 | ❌ | nrfutil URL 404 (old `linux-amd64`) | — |
| #2 | 24655705928 | ❌ | same (triggered by Day 5 push) | — |
| #3 | 24656054715 | ❌ | `git: 'remote-https' is not a git command` (env flatten not clean enough) | `e4cc3f4` URL+cache+env |
| #4 | 24656211384 | ❌ | `git-remote-https: libunistring.so.2: cannot open` | `a9b33d4` launch sandbox |
| #5 | **24656395010** | ✅ **7m7s** | — | `5dc9435` libunistring symlink |

Key insight: the NCS toolchain bundled git is linked against `libunistring.so.2`, but Ubuntu 24.04 (noble) ships only `.so.5`. ABI-compatible — a symlink is enough. This is the same gotcha we hit locally on Day 5, where `source /tmp/ncs_env.sh + unset LD_LIBRARY_PATH` sidestepped it; in CI we cannot sidestep (NCS git is what does `west update`), so we had to fix it head-on.

**Final workflow shape**:
- Every NCS command wrapped in `nrfutil toolchain-manager launch --shell -- <cmd>` (env isolation)
- apt stage creates the symlink `libunistring.so.2 -> libunistring.so.5` (ABI shim)
- Cache key bumped to `-v2` to invalidate old broken cache

### Key learnings
1. **CI is the second validation shot at Day 5's learnings**: the workflow was written on Day 4 on macOS from memory; after the user moved to Linux on Day 5, local stepping on the same rakes also exposed the same bugs in the workflow. Fixing the workflow = baking local experience into automated regression.
2. **Fatal flaw of `curl -sL`**: `-s` silent + `-L` follow redirects, but does **not** fail on HTTP errors. Future default for all CI scripts: `curl -fsSL` so HTTP 4xx/5xx exits immediately.
3. **Cache-key strategy**: bake platform-diff paths into a key suffix (like `-v2`) so old caches invalidate gracefully, no need to manually clear GitHub Actions cache.
