## Execution log

### Task 1: GitHub Actions Workflow
File: `.github/workflows/build.yml`

Design:
- Ubuntu 22.04 runner
- **Two-level cache**:
  - `ncs-toolchain-v2.9.0-Linux` — caches `/opt/nordic/ncs/toolchains/` (1.5 GB)
  - `ncs-modules-v2.9.0-<hash(west.yml)>` — caches `nrf/ zephyr/ nrfxlib/ modules/ bootloader/ tools/ test/ .west/` (hundreds of MB)
  - On cache hit, skips `nrfutil toolchain-manager install` and `west update`; expected to drop from 15+ min to ~3 min
- **NCS env propagation**: write `nrfutil ... --as-script` output to a file and source it, then extract `PATH/ZEPHYR_*/GNUARMEMB` etc. into `$GITHUB_ENV` so later steps don't need to source again
- **Dual-build strategy**: first run a regular sysbuild (ensure no normal regression), then the `-Werror` strict build (AC-Q1), then the `verify-acceptance.sh` assertions
- **Artifact upload**: `merged.hex` as a CI artifact so the interviewer can download-and-verify per commit

Risks:
- Cannot locally verify whether the `nrfutil linux-amd64` binary is downloadable on a GitHub-hosted Ubuntu runner (only visible after push). Day 5 first item: look at the Actions tab — fix per log if red.
- `west update --narrow -o=--depth=1` speeds up the initial clone — the combo is officially recommended for NCS.

### Technical notes
- **CI cache key design**: modules use `hashFiles('west.yml')` so any west.yml change invalidates automatically; toolchain uses a fixed key (invalidated only when NCS version changes), because toolchain download is the slowest.
- **sysbuild and Twister**: `sysbuild: true` in `sample.yaml` makes Twister follow the same path as `--sysbuild` (otherwise the net core is not built).
- **--depth=1 risk**: shallow clone occasionally fails because an NCS-dependent manifest tag is deep. If `west update` fails in CI, first response is to drop `--narrow -o=--depth=1` and go full clone.
- **Value of the handoff doc**: not for the interviewer, but for "tomorrow's self on a strange machine". The four days of implicit context (why `path: nrf` in west.yml, why `CONFIG_BT_HCI_IPC` cannot be set, why zsh has trouble with `eval "$(...)"`) made fully explicit — no stepping on the same rakes twice.
